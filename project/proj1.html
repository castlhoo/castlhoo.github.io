Project

Chicago Crime Risk Prediction & AI Insurance Chatbot

2025 Personal Project

Overview

This project is an end-to-end AI system that predicts regional crime risk from large-scale public safety data and translates those predictions into actionable insurance recommendations through a conversational interface. Rather than stopping at model prediction, I designed the pipeline to connect data engineering, feature engineering, model optimization, and AI deployment into one integrated system.

Problem Statement

Public crime datasets are complex and difficult for non-technical users to interpret. The goal was to not only predict crime risk, but also convert those predictions into intuitive, user-friendly insights and personalized insurance recommendations using a GPT-based chatbot (InsureWise).

Project Walkthrough

1. System Architecture

The full pipeline: Kaggle crime data → PySpark (on Colab) for preprocessing and feature engineering → ML models (LightGBM, Random Forest, XGBoost) → Tableau for visualization → Flask API + GPT-powered chatbot (InsureWise) for end-user interaction. The right side shows the actual InsureWise chatbot interface, where users select their area and housing type to receive personalized crime risk analysis and insurance recommendations.

2. Data Preprocessing

I processed approximately 237,000 crime records using PySpark:

  • Location categorization — Mapped raw location descriptions into structured categories (Residential, Commercial, Transport, Public, Education, Medical, etc.)
  • Crime type classification — Grouped primary types into categories (Theft → Property, Battery → Violent, Narcotics → Drug, Weapons Violation → Weapons, etc.)
  • Temporal features — Added Season (Winter/Spring/Summer/Fall) and Day Type (Weekday/Weekend) classifications

3. Feature Engineering

I engineered over 100 features and designed three core risk scoring systems:

  • Area Risk Score — Calculated per community area by normalizing crime counts (groupBy CA_Name)
  • Crime Severity Score — Weighted scoring by crime type (Homicide: 10, Criminal Sexual Assault: 9, Kidnapping: 8, Robbery: 7, Assault: 6, Burglary: 4, Theft: 3)
  • Total Risk Score — Composite metric combining Crime Severity × Location Risk using PySpark UDFs, creating a single risk indicator per record

4. Modeling and Evaluation

I compared three models using accuracy, F1 score, validation performance, and overfitting metrics:

  • XGBoost (1st) — 89.96% test accuracy, 89.05% F1, 89.76% validation accuracy, 0.80% overfitting, 29.04s training
  • Random Forest (2nd) — 89.24% accuracy, 87.64% F1, 88.73% validation, 0.69% overfitting, ~60s training
  • LightGBM (3rd) — 76.02% accuracy, 80.01% F1, 76.02% validation, -0.15% overfitting, 2.02s training

XGBoost was selected as the final model — best accuracy with strong generalization and minimal overfitting.

5. Flask API with GPT Chatbot

I deployed the trained XGBoost model as a Flask API with two core endpoints:

  • /predict endpoint — Takes user location data (name, region, housing type, community area), runs the model, and returns crime risk predictions
  • /gpt_recommend endpoint — Feeds the prediction results into GPT, which generates personalized insurance recommendations in natural language, mapping housing types to readable labels (Residential → Apartment/House, Commercial → Store/Office, etc.)

This transforms the project from a prediction model into a user-facing AI application (InsureWise).

6. Analytical Dashboard

The Tableau dashboard visualizes the final analytical output with four components:

  • Crime in Chicago (bubble map) — Regional crime distribution showing West Side and South Side as highest-risk areas
  • Crime analysis in location — Bar chart breaking down crimes by location category (Residential dominant)
  • Crime category in region — Table showing Violent, Weapons, Drug, Property, and Sex Crime counts per region
  • Crime severity — Horizontal bar chart comparing severity across crime categories (Violent as highest)

What I Gained

  • Large-scale data processing with PySpark — Processed 237K records with structured categorization, temporal feature extraction, and custom UDFs, learning how to handle data at scale beyond pandas.
  • Designing interpretable risk scoring systems — Built composite risk metrics (Area Risk × Crime Severity) that are not just model inputs but also directly meaningful to end users and insurance professionals.
  • Rigorous model comparison methodology — Went beyond just accuracy to evaluate overfitting, validation performance, and training efficiency across three models to make an informed deployment decision.
  • Connecting ML to real user experience — Learned that the most impactful part of an ML project is often the last mile: turning predictions into something a non-technical user can understand and act on through GPT-powered natural language.
  • Full-stack AI system design — Built the complete path from raw data to deployed chatbot (Kaggle → PySpark → XGBoost → Flask → GPT → InsureWise), understanding how each layer connects in a production-oriented AI system.
