Project

Chicago Crime Risk Prediction & AI Insurance Chatbot

2025 Personal Project

Overview

This project is an end to end AI system that predicts regional crime risk from large scale public safety data and converts those predictions into practical insurance recommendations through a conversational interface. I designed the workflow to connect data engineering, machine learning, and AI application development in one pipeline, moving from raw Chicago crime data to a user facing GPT based recommendation system. :contentReference[oaicite:1]{index=1}

Problem Statement

Public crime datasets are difficult for most users to interpret directly. I wanted to build a system that not only estimates neighborhood level crime risk, but also explains that risk in a more accessible way and translates it into actionable insurance guidance. The final result was a prediction pipeline paired with an AI chatbot that helps users understand local risk and receive more personalized recommendations. :contentReference[oaicite:2]{index=2}

Final Product View

This dashboard view represents the final analytical output of the project. It combines regional crime distribution, area level risk visualization, category level crime patterns, and severity analysis into one interface. The goal here was to make the model output interpretable, not just accurate, so users and decision makers could immediately see which areas and crime categories were driving risk. :contentReference[oaicite:3]{index=3}

System Architecture

This architecture diagram shows the full system flow. I started from the Kaggle crime dataset and used Tableau for exploratory analysis and Spark based modeling for scalable learning. After comparing multiple machine learning models, I exposed the best model through a Flask API and connected it with a GPT based chatbot so users could interact with the system through natural language rather than raw prediction tables. This project was intentionally built as both a data science workflow and an AI product prototype. :contentReference[oaicite:4]{index=4}

Data Preprocessing

I began with a Chicago crime dataset containing about 237 thousand records. Using PySpark, I cleaned the data, handled missing values, and standardized important fields such as crime type, location category, and timestamp information so the dataset could be used reliably for modeling. I also created normalized categorical structures that made later feature generation and model training much more consistent. :contentReference[oaicite:5]{index=5}

Feature Engineering

After cleaning the dataset, I engineered more than 100 features. These included temporal patterns such as hour of day, weekday, and season, as well as geographic risk indicators and crime severity related variables. In particular, I designed area risk scores, location risk features, and severity based scoring logic so that the model could capture both where crimes happen and how serious those crimes are. This step was one of the most important parts of the project because it transformed raw event data into structured signals that meaningfully improved predictive power. :contentReference[oaicite:6]{index=6}

Modeling and Evaluation

I trained and compared multiple models including XGBoost, Random Forest, and LightGBM. Among them, XGBoost achieved the strongest overall performance with 89.96% test accuracy, 89.05% test F1, and 89.76% validation accuracy. I also compared the models in terms of overfitting and training time so that the final selection balanced predictive strength with practical deployment considerations. This made the project more than a pure leaderboard exercise because the final model needed to be both accurate and usable in an application setting. :contentReference[oaicite:7]{index=7}

AI Application Layer

Once the model was selected, I deployed it as a Flask API and integrated it with a GPT based chatbot. The API accepts user information such as region and housing type, runs the underlying crime risk prediction, and returns structured output that the GPT layer can turn into a more natural recommendation. This means the system does not simply output a score. Instead, it explains risk in plain language and links that risk to personalized insurance suggestions, which made the project feel much closer to a real product experience. :contentReference[oaicite:8]{index=8}
