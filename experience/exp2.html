Overview

At UC San Diego MixLab, I work on LLM systems for scientific reproducibility. My work focuses on building a pipeline that connects scientific papers, structured experimental datasets, and evaluation driven feedback so that large language models can reproduce key analyses from published research.

Core Focus

The project is centered on one main question: can LLMs understand scientific papers and experimental data well enough to reproduce meaningful research results. To answer this, I worked on four core components: structuring scientific datasets, designing task grounded prompts, benchmarking model outputs, and improving results through a RAG based feedback loop.

Key Responsibilities

  • Structured multimodal scientific datasets (Raman spectroscopy, AFM, electronic transport) into annotation, figure level, and raw data layers for LLM consumption.
  • Designed task grounded prompts that guide LLMs through reproducible scientific workflows including data loading, variable identification, and plot reconstruction.
  • Built a multi dimensional benchmarking framework to evaluate model outputs against original paper figures across scientific consistency, methodology, and visualization quality.
  • Implemented a RAG pipeline using ChromaDB to parse, chunk, and retrieve relevant sections from scientific papers during analysis.
  • Developed an evaluation driven repair loop that detects weaknesses in model outputs and iteratively improves them using retrieved context.

Achievements

  • Successfully reproduced key analyses from published research papers using LLM generated code and reasoning, validated against original figures.
  • Demonstrated measurable improvement in scientific accuracy after applying RAG based retrieval and iterative feedback compared to one shot generation.
  • Built a reusable end to end pipeline (data structuring → prompting → evaluation → retrieval → repair) that can be applied to new scientific domains beyond the initial dataset.

Project Walkthrough

Project Objective

The main objective of the UC LEAP project is to evaluate whether large language models can understand both scientific papers and experimental datasets well enough to reproduce key analyses. My contribution focuses on making that question testable through structured tasks, grounded prompts, measurable evaluation, and iterative improvement.

1. Structured Data Representation

I first organized the dataset into clear layers such as annotation, figure level data, and raw data so the model could work with a consistent scientific structure instead of unorganized files. I also defined required inputs and relevant variables such as filling factor, magnetic field, and inverse compressibility so each task was grounded in explicit experimental targets. This step was important because it turned the dataset into a machine readable research workspace rather than a loose collection of files.

2. Prompt Engineering

After structuring the dataset, I designed step by step prompts that force the model to follow a reproducible scientific workflow. Instead of asking the model to solve the task directly, the prompt guides it through data loading, variable identification, inspection, and plot reconstruction in sequence. This improved performance compared with less structured prompting and reduced failure caused by vague reasoning.

3. Benchmarking and Evaluation

I built an evaluation framework to score each model output against the original paper figures. The framework measures multiple dimensions such as scientific consistency, methodology correctness, data handling completeness, visualization quality, and analysis depth. This allowed me to compare models quantitatively, track token efficiency, and identify which agents performed better on which tasks instead of relying on subjective inspection alone.

4. RAG Pipeline for Scientific Context

I then built a RAG pipeline that parses scientific papers, chunks the text, and stores it in a vector database for retrieval. This gives the model access to targeted scientific context during analysis rather than relying only on internal memory. In practice, this lets the system retrieve the most relevant paper sections, variables, and methodological details when reconstructing experiments.

5. Weakness Analysis from Evaluation

Once evaluation scores were available, I used them to detect specific weaknesses in model outputs. Rather than saying an output was simply wrong, I extracted which dimension failed, such as shallow interpretation or incomplete handling of the dataset. This converted evaluation into a diagnostic tool and made the next improvement step much more targeted.

6. RAG based Repair and Iterative Improvement

After identifying weak points, I built a repair pipeline that combines retrieved context with evaluation driven feedback to improve the original output. The system retrieves evidence relevant to the detected weakness, then uses that information to revise reasoning and code rather than generating from scratch again. This creates an iterative loop where the model improves based on grounded feedback instead of one shot trial and error.

7. Result Comparison

The final comparison shows the practical value of the pipeline. Before RAG and feedback, generated plots often missed important physical patterns from the paper. After retrieval and repair, the outputs aligned much more closely with the original figures, showing that grounded context and iterative correction meaningfully improved scientific accuracy and reproducibility.

Research Summary

At a system level, this work combines model generation, evaluation, retrieval, and feedback into one research pipeline. Different models generate analyses from the same scientific dataset, the outputs are scored against paper level references, and the results are improved through a feedback loop. The broader goal is to build AI systems that can move beyond summarizing papers and instead reproduce scientific reasoning in a grounded and testable way.

What I Gained

  • Designing LLM systems for real scientific tasks — Learned how to move beyond generic prompting and build task specific pipelines where LLMs interact with structured data, follow scientific workflows, and produce verifiable outputs.
  • End to end pipeline thinking — Experienced the full cycle from raw data structuring to evaluation to retrieval to iterative repair, understanding how each component feeds into the next.
  • RAG architecture for domain specific retrieval — Built a retrieval system tailored for scientific papers, learning how chunking strategy, embedding quality, and context relevance directly affect downstream model performance.
  • Evaluation as a diagnostic tool — Shifted from treating evaluation as a pass or fail metric to using it as a structured signal that drives targeted improvement in model outputs.
  • Working with multimodal scientific data — Gained hands on experience with Raman spectroscopy, AFM imaging, and electronic transport measurements, learning to bridge the gap between domain expertise and AI system design.
